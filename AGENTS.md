## 프로젝트 개요

**REBE** (Real Bridge, "리얼브릿지")는 **부동산·금융 업계 취업컨설팅** 서비스입니다.
500명 이상의 **현직 실무자(멘토) 네트워크**를 활용해 지원 회사·직무의 현직자를 직접
연결하고, 자기소개서·면접·커리어 방향성을 컨설팅하며, 비공개 채용정보와 실무 꿀팁을
제공합니다.

**이 프로젝트의 목적**: 이미 운영 중인 서비스(현재 크몽 등 외부 플랫폼에서 판매)를
**자체 홈페이지로 이전**하는 것. 외부 플랫폼 **수수료를 절감**하고 브랜드를 직접
운영하는 것이 핵심 동기입니다.

**핵심 사용자 흐름**: 서비스 둘러보기 → 멘토/패키지 확인 → 상담 신청 또는 회원가입
→ 채용정보·콘텐츠 열람 → 컨설팅 패키지 결제

## 기술 스택

- **프레임워크**: Next.js 16 (App Router) + React 19 + TypeScript
- **스타일링**: Tailwind CSS v4
- **데이터베이스 / 인증**: Supabase (PostgreSQL + Auth + Storage)
- **결제**: 토스페이먼츠 (TossPayments)
- **배포**: Vercel

> ⚠️ Next.js 16은 이전 버전과 차이가 큽니다. 코드 작성 전 `node_modules/next/dist/docs/01-app/`의 해당 문서를 확인하세요.

## 주요 기능

1. **서비스 / 회사 소개** — REBE 소개, 컨설팅 패키지(STANDARD/DELUXE/PREMIUM), 진행 방식
2. **멘토 소개** — 현직 실무자(멘토) 프로필, 소속 산업군·직무, 네트워크 강점 강조
3. **상담 신청 / 문의 폼** — 입력 검증 후 저장 + 알림(이메일 등)
4. **채용정보 / 취업 꿀팁 게시판** — 부동산·금융 채용정보, 실무 팁 글 목록·상세, SEO 최적화
5. **회원가입 / 로그인** — 인증 기반 보호 페이지(비공개 자료실, 마이페이지)
6. **결제 처리** — 컨설팅 패키지 결제, 결제 상태 관리, 영수증
7. **후기 / 실적** — 컨설팅 후기, 합격 사례, 신뢰 지표 노출

## 코딩 규칙

- **언어**: UI 문구는 한국어, 코드 식별자는 영어.
- **타입**: `any` 지양. 공용 타입은 `types/`에 정의하고 재사용.
- **컴포넌트**: 가능한 한 서버 컴포넌트 우선. 상호작용이 필요할 때만 `"use client"`.
- **스타일**: Tailwind 유틸리티 클래스 사용. 임의 인라인 스타일 지양.
- **폼**: 클라이언트·서버 양쪽에서 검증. 서버 검증을 신뢰의 기준으로 둠.
- **파일명**: 컴포넌트는 PascalCase, 그 외는 kebab-case.

## 보안 / 개인정보 (중요)

- 상담 신청자의 **개인정보(이름·연락처·이메일)는 민감 데이터**입니다. 로그에 남기지 마세요.
- **API 키·DB 비밀번호·결제 시크릿은 절대 코드/저장소에 하드코딩하지 않습니다.** `.env.local`만 사용하고 `.gitignore`에 포함하세요.
- 결제 검증은 **반드시 서버 측에서** 처리합니다. 클라이언트가 보낸 금액·결제 성공 여부를 그대로 신뢰하지 않습니다.
- 인증 보호 페이지는 미들웨어 또는 서버에서 세션을 확인합니다.
- 개인정보처리방침·이용약관 페이지를 갖추고, 폼 제출 시 동의를 받습니다.

- 결제 전 문의 창구로 **채널톡(channel.io)** 위젯 연동. `components/channel-talk.tsx`(클라이언트)가

- 운영: 채널톡 모바일/웹 앱에서 문의 응대, 자동응답·영업시간 설정. 패키지: `@channel.io/channel-web-sdk-loader`.

## 법적 표현 가이드 (마케팅 문구 작성 시 준수)

- **"비공개/내부/오픈되지 않은 채용정보를 제공한다"는 식의 표현 금지.** 영업비밀·현직자
  비밀유지의무 위반 소지. → "현직자 경험 기반 직무·업계 동향", "채용공고에 드러나지 않는
  핵심 역량·직무 인사이트" 등으로 순화.
- **특정 기업 실명을 '멘토 소속/네트워크'로 나열 금지** (상표·제휴 오인·내부정보 암시).
  → 기업 카테고리("국내외 자산운용사", "대형 증권사" 등)로 표기.
- **합격 사례 기업 실명**은 사실 적시 범위에서 사용하되, 반드시 **비제휴·비보증 면책 문구**
  병기. 기업이 이의 제기 시 마스킹(예: "S사") 전환 검토.
- **수치 광고(합격률·만족도 등)**는 과장·기만 소지(표시광고법). 근거 보유 + "자체 집계 기준"
  면책 병기. 근거 없는 단정 표현 지양.
- 최종 문구는 변호사 검수 권장. 위 원칙은 리스크 완화용 기본선.

## 작업 시 유의사항

- 작은 단위로 변경하고, 기존 패턴이 있으면 그것을 따릅니다.
